# @identity-backend/play-integrity

Android app attestation service using Google Play Integrity API.

## What it does

Verifies that requests to your backend are coming from legitimate Android apps installed through Google Play. This helps protect against:

- Modified or tampered apps
- Apps running on compromised devices
- Automated bot attacks
- Token replay attacks

## When to use

Use this package when you need to ensure your backend only accepts requests from genuine versions of your Android app. Common scenarios:

- Protecting sensitive API endpoints
- Preventing abuse of premium features
- Securing in-app purchases
- Validating high-value transactions

## How it works

1. Your Android app requests an integrity token from Google Play
2. The app sends this token to your backend
3. This service verifies the token with Google's servers
4. You receive attestation details about the app and device
5. Your backend decides whether to trust the request

## Setup Requirements

1. Google Cloud service account with Play Integrity API enabled
2. Android app configured with Play Integrity in Google Play Console
3. Effect-TS runtime environment

## Key Concepts

**Integrity Token**: Single-use cryptographic proof generated by Google Play on the device

**Verdict**: Google's assessment of the app and device trustworthiness

**App Recognition**: Confirms the app matches the version distributed through Google Play
